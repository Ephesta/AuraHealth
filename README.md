# This project is developed for NYP class.
# I trained a machine learning model using dataset I collected from the online sources.The dataset I collected included one year of weather data for all 81 provinces.
# After training the model and verifying its accuracy, I started working on integrating it with the Gemini API.
# Initially, I deployed my model using Flask and made it accessible locally via Ngrok for testing purposes. The tests were successful
# Then I moved on to the stage of making my model usable through the Gemini API, and this part took a bit longer.
# The basic idea was that when a weather prediction query was sent to Gemini, it would return the output generated by my ML model.
# After that i coded a web interface with html/css/js. Actually, I wanted to do this project as a mobile application, but I have absolutely no knowledge of mobile programming.
# After building the web interface, I took a short break from my journey with the model and Gemini. During that time, I used the OpenWeatherMap API to display real-time weather and air pollution data (also available based on the user's location). Additionally, I fetched earthquake data from Kandilli Rasathanesi by performing web scraping.
# Then I returned to my journey and improved my app.py file. By "improved," I mean that previously, Gemini was detecting weather-related future questions within sentences and responding accordingly. However, this approach made the code extremely cluttered — it had practically turned into a pile of if-else statements. So I abandoned that idea and instead added a command where the user could enter the location and date for which they wanted a forecast. This approach turned out to be much simpler and better — at least in my opinion.
# At this point, I can't think of much else I've done, but I believe my project still has a lot of room for improvement. Gemini can't handle long conversations — it quickly goes off track. My model was trained with only one year of data, which I think is too little for accurate weather forecast. Honestly, I could have collected more, but since I don’t claim to be great at training models, I wanted to avoid issues like overfitting.
# And finally, I know I could have added something like user identity or profiles, but there's really no feature in the app that requires user identification — users don't chat with each other or need accounts. Also, I’m aware that using API keys directly in the code is a security issue, but this project is still at a very basic level, far from the point where I need to worry about security concerns.
